# QTE Bot Auto — Автоматичний бот для QTE міні-ігор

Бот для автоматичного проходження QTE міні-ігор у 3D-іграх на базі DirectX.
Коли на екрані з'являється **будь-яка літера (A–Z)**, бот миттєво її розпізнає через Tesseract OCR і натискає відповідну клавішу.

---

## 📦 Крок 1: Встановлення Python-залежностей

Відкрийте термінал (PowerShell) у папці проєкту та виконайте:

```bash
pip install mss numpy opencv-python pytesseract pydirectinput keyboard
```

| Бібліотека | Для чого |
|---|---|
| `mss` | Швидке захоплення екрана |
| `opencv-python` | Обробка зображень (бінаризація, інверсія, padding) |
| `numpy` | Робота з масивами пікселів |
| `pytesseract` | Python-обгортка для Tesseract OCR |
| `pydirectinput` | Натискання клавіш через DirectInput (для DirectX ігор) |
| `keyboard` | Глобальні гарячі клавіші (F8, F9) |

---

## 🔤 Крок 2: Встановлення Tesseract OCR (ОБОВ'ЯЗКОВО)

`pytesseract` — це лише Python-обгортка. Щоб OCR працював, потрібно встановити сам **Tesseract OCR Engine** на вашу систему.

### Покрокова інструкція:

**1.** Завантажте інсталятор для Windows з офіційного репозиторію UB-Mannheim:

> 🔗 **https://github.com/UB-Mannheim/tesseract/wiki**

На сторінці знайдіть розділ **"The latest installers can be downloaded here"** і скачайте файл `tesseract-ocr-w64-setup-v5.x.x.exe` (64-bit).

**2.** Запустіть інсталятор:

- Натисніть **Next** на кожному кроці.
- **Шлях встановлення** залиште стандартний:
  ```
  C:\Program Files\Tesseract-OCR
  ```
- На кроці **"Select Components"** переконайтесь, що вибрано хоча б `English` у розділі мов. Додаткові мови не потрібні.
- Натисніть **Install**, дочекайтесь завершення.

**3.** Перевірте, що Tesseract встановився правильно. Відкрийте PowerShell і виконайте:

```powershell
& "C:\Program Files\Tesseract-OCR\tesseract.exe" --version
```

Ви повинні побачити щось на кшталт:
```
tesseract v5.5.1
```

**4.** Якщо ви встановили Tesseract **за іншим шляхом** (не `C:\Program Files\Tesseract-OCR`), відкрийте файли `qte_bot_auto.py` та `test_bot.py` і змініть рядок:

```python
pytesseract.pytesseract.tesseract_cmd = r"C:\Program Files\Tesseract-OCR\tesseract.exe"
```

на ваш фактичний шлях, наприклад:

```python
pytesseract.pytesseract.tesseract_cmd = r"D:\Tools\Tesseract-OCR\tesseract.exe"
```

## ⚙️ Крок 3: Налаштування координат екрана (КРИТИЧНО)

Перед запуском **обов'язково** вкажіть координати зони на екрані, де з'являється літера у грі.

Відкрийте `qte_bot_auto.py` і знайдіть рядок:

```python
MONITOR = {"top": 400, "left": 800, "width": 150, "height": 150}
```

### Що означають ці значення:

```
┌────────────────────────────────────────────────┐
│                    Ваш екран                    │
│                                                │
│         left                                   │
│          ↓                                     │
│   top → ┌──────────┐                           │
│         │  Літера   │ ← height                 │
│         │    (W)    │                           │
│         └──────────┘                            │
│            width                               │
│                                                │
└────────────────────────────────────────────────┘
```

- **`top`** — відступ від верхнього краю екрана (пікселі)
- **`left`** — відступ від лівого краю екрана (пікселі)
- **`width`** і **`height`** — розмір зони захоплення (пікселі)

### Як знайти правильні координати:

1. Зайдіть у гру та дочекайтесь появи QTE (кружечок з літерою).
2. Натисніть **`Win + Shift + S`** і виділіть область навколо кружечка.
3. Відкрийте скріншот у **Paint**:
   - Наведіть курсор на **лівий верхній кут** кружечка — координати відображаються внизу вікна. Це ваші `left` та `top`.
   - Виміряйте ширину та висоту кружечка — це `width` та `height`.

> **Порада:** Робіть зону **якомога меншою** (100–200 px). Чим менша область, тим швидше працює OCR і тим менше шансів зловити зайвий текст.

## 🧪 Крок 4: Тестування (перед грою)

Перш ніж запускати бота у грі, переконайтесь, що OCR працює правильно.

### Варіант А: Без скріншотів (автоматичний тест)

Просто запустіть тестовий скрипт — він автоматично згенерує 26 тестових зображень (A–Z) і перевірить розпізнавання кожного:

```bash
python test_bot.py
```

Очікуваний результат:
```
Файл                      Результат       Статус
-------------------------------------------------------
qte_a.png                 A               ✅
qte_b.png                 B               ✅
...
qte_z.png                 Z               ✅

📊  ПІДСУМОК: Розпізнано 25/26 зображень
```

> **Примітка:** Літера **«I»** може не розпізнатися на автоматично згенерованих зображеннях, тому що шрифт OpenCV малює її як тоненьку палочку без засічок. В реальній грі шрифт значно товщіший і має засічки — **«I» розпізнається коректно**. Якщо ваш тест зі скріншотом з гри теж не бачить «I» — зменшіть `THRESHOLD_VALUE` до `100`.

### Варіант Б: Зі скріншотами з гри

1. Зробіть скріншоти QTE-кружечка у грі (`Win + Shift + S`).
2. Збережіть їх у папку `test_images/` поруч із скриптами.
3. Запустіть `python test_bot.py`.

---

## 🎮 Крок 5: Запуск у грі

```bash
python qte_bot_auto.py
```

### Гарячі клавіші:

| Клавіша | Дія |
|---------|-----|
| **F8** | Увімкнути / Призупинити сканування |
| **F9** | Повністю закрити програму |

### Порядок роботи:

1. Запустіть скрипт → побачите **"Бот ГОТОВИЙ!"**.
2. Перейдіть у гру, починайте міні-гру.
3. Натисніть **F8** — бот почне сканувати екран і натискати літери.
4. Коли міні-гра закінчилась — натисніть **F8** ще раз (пауза).
5. Щоб повністю закрити бота — натисніть **F9**.

> **Важливо:** Завжди ставте бота на паузу (F8), коли QTE не активне!

## 🔧 Troubleshooting (Вирішення проблем)

### ❌ "Tesseract OCR не знайдено"

Перевірте, чи файл `tesseract.exe` існує за вказаним шляхом:
```powershell
Test-Path "C:\Program Files\Tesseract-OCR\tesseract.exe"
```
Якщо `False` — перевстановіть Tesseract (Крок 2) або вкажіть правильний шлях.

---

### ❌ Бот не розпізнає літеру

**Причина 1: Поріг бінаризації занадто високий.**

У `qte_bot_auto.py` знайдіть:
```python
THRESHOLD_VALUE = 140
```
Зменшіть до `100` або `80`:
```python
THRESHOLD_VALUE = 100
```

**Причина 2:** Зона `MONITOR` захоплює занадто багато елементів інтерфейсу. Зменшіть `width` та `height`.

---

### ❌ Бот не натискає клавіші в грі

1. Запустіть скрипт **від імені адміністратора**.
2. Переконайтесь, що гра у **віконному** або **безрамковому** режимі (Borderless Windowed), а не в Exclusive Fullscreen.
