# QTE Bot Auto — Автоматичний бот для міні-ігор (QTE)

Бот для автоматичного проходження QTE міні-ігор у 3D-іграх на базі DirectX (наприклад, Ukraine GTA).  
Коли на екрані в колі з'являється літера (**W**, **A**, **S** або **D**), бот миттєво її розпізнає і натискає відповідну клавішу.

> **Принцип роботи:** бот НЕ використовує OCR чи нейромережі. Він генерує **480 варіантів шаблонів** (5 шрифтів × 6 товщин × 4 масштаби × 4 літери) та порівнює геометричну форму знайденого контуру з кожним із них за допомогою алгоритму **Hu Moments (I3)** в OpenCV. Це забезпечує точне розпізнавання незалежно від шрифту гри.

## 📦 Встановлення

### 1. Python

Потрібен **Python 3.10+**. Перевірте версію:
```bash
python --version
```

### 2. Залежності

Відкрийте термінал у папці проєкту та виконайте:
```bash
pip install mss numpy pydirectinput keyboard opencv-python
```

| Бібліотека | Для чого |
|---|---|
| `mss` | Швидке захоплення екрана (значно швидше за стандартний `ImageGrab`) |
| `opencv-python` | Обробка зображень: бінаризація, пошук контурів, порівняння форм |
| `numpy` | Робота з масивами пікселів |
| `pydirectinput` | Емуляція клавіатури через DirectInput (працює у DirectX іграх, де `pyautogui` ігнорується) |
| `keyboard` | Глобальні гарячі клавіші для керування ботом |

## ⚙️ Налаштування (КРИТИЧНО)

Перед першим запуском **обов'язково** вкажіть координати зони на екрані, де з'являється літера у грі.

Відкрийте файл `qte_bot_auto.py` та знайдіть **рядок 12**:

```python
MONITOR = {"top": 400, "left": 800, "width": 150, "height": 150}
```

### Що означають ці значення

```
┌────────────────────────────────────────────────┐
│                    Ваш екран                    │
│                                                │
│         left                                   │
│          ↓                                     │
│   top → ┌──────────┐                           │
│         │  Літера   │ ← height                 │
│         │    (W)    │                           │
│         └──────────┘                            │
│            width                               │
│                                                │
└────────────────────────────────────────────────┘
```

- **`top`** — відступ від верхнього краю екрана (у пікселях)
- **`left`** — відступ від лівого краю екрана (у пікселях)
- **`width`** та **`height`** — розміри зони захоплення

### Як дізнатися правильні координати

1. Зайдіть у гру і дочекайтесь появи QTE (кружечок з літерою).
2. Натисніть `Win + Shift + S` (стандартний інструмент Windows) і виділіть область навколо кружечка.
3. Відкрийте скріншот у Paint і наведіть курсор на **лівий верхній кут** кружечка — координати будуть видно внизу вікна. Це ваші `left` та `top`.
4. Виміряйте ширину та висоту кружечка — це `width` та `height`.

> **Порада:** Робіть зону якомога **меншою** (100–200 px). Чим менша область — тим швидше працює бот і тим менше шансів зловити зайвий текст (чат, нікнейми).

## 🎮 Запуск у грі

### 1. Запустіть бота

```bash
python qte_bot_auto.py
```

### 2. Гарячі клавіші

| Клавіша | Дія |
|---------|-----|
| **F8** | Увімкнути / Призупинити сканування |
| **F9** | Повністю закрити програму |

### 3. Порядок роботи

1. Запустіть скрипт → побачите напис **"Бот ГОТОВИЙ!"**.
2. Заходьте у гру, починайте міні-гру.
3. Натисніть **F8** — бот почне сканувати екран.
4. Коли міні-гра закінчилась — натисніть **F8** ще раз (пауза), щоб бот не працював вхолосту.
5. Щоб повністю закрити — натисніть **F9**.

> **Важливо:** Завжди ставте бота на паузу (F8), коли міні-гра не активна. Інакше він може випадково натиснути клавішу під час звичайної гри.

## 🔧 Troubleshooting (Вирішення проблем)

### Бот нічого не розпізнає (не натискає клавіш)

**Причина 1: Поріг бінаризації занадто високий.**

У файлі `qte_bot_auto.py`, рядок з `THRESHOLD_VALUE`:
```python
THRESHOLD_VALUE = 140
```

Число `140` — це поріг яскравості. Все, що яскравіше за 140, стає білим; все темніше — чорним. Якщо літера у грі напівпрозора або сіра, вона може не пройти цей поріг.

**Рішення:** Зменшіть `140` до `100` або навіть `80`:
```python
THRESHOLD_VALUE = 100
```

---

**Причина 2: Поріг збігу занадто суворий.**

У файлі `qte_bot_auto.py`, рядок з `MATCH_SCORE_LIMIT`:
```python
MATCH_SCORE_LIMIT = 0.25
```

Число `0.25` означає: "якщо форма контуру відрізняється від ідеалу менш ніж на 0.25, вважаємо це збігом". Якщо ігровий шрифт дуже нестандартний, збіг може бути `0.30`, і бот його проігнорує.

**Рішення:** Послабте ліміт до `0.35`:
```python
MATCH_SCORE_LIMIT = 0.35
```

> **Увага!** Не ставте значення вище `0.5` — інакше бот почне реагувати на випадкові елементи інтерфейсу.

---

### Бот натискає не ту літеру

**Причина:** Зона `MONITOR` захоплює зайві елементи (текст чату, смужку прогресу, елементи інтерфейсу).

**Рішення:** Зменшіть `width` та `height` у `MONITOR` так, щоб у зону потрапляв **тільки кружечок з літерою**.

---

### `pydirectinput` не натискає клавіші в грі

1. Запустіть скрипт **від імені адміністратора** (правою кнопкою → "Запустити від імені адміністратора").
2. Переконайтесь, що гра у **віконному** або **безрамковому** режимі (не повноекранний Exclusive Fullscreen).
